<Road Balancing>

1. 서버에 들어오는 네트워크 요청이나 작업 부하(트래픽)를 여러 대의 서버에 고르게 분산하는 기술.

2. 로드 밸런싱이 필요한 이유.
    단일 서버에 모든 요청이 집중되면 과부하가 걸려 느려지고, 장애가 생길 수도 있음

    여러 서버에 부하를 나누면 서비스 안정성과 처리 속도가 좋아짐

    서버가 다운돼도 다른 서버가 대신 처리할 수 있어 가용성(서비스 연속성)이 높아짐

3. 같은 기능을 하는 서버 여러개를 두고 있어야 한다.

4. 같은 기능을 하는 서버 여러개가 동시에 돌고 있고 로드밸런서를 통해서 골고루 분배된다.

5. 서버 다중화랑은 다른 개념.
    서버 다중화는 서버하나만 사용하다가 서버 한대가 고장나면 다른 서버로 바꿔 끼는 느낌이라면
    로드 밸런싱은 서버 여러개가 같이 움직이고 있고 밸런서에 의해 골고루 일을 나눠 받는다.
